## 3. Metas Técnicas (Como pretendo evoluir)

### Meta 1: Análise de Performance e Big O
* **Contexto:** Essencial para avaliar o impacto de novas rotinas de análise de crédito na NeoCredit.
* **Descrição:** Realizar a análise de complexidade de tempo ($O(n)$) de todos os códigos desenvolvidos para as listas de exercícios.
* **Plano de ação:** Estudar profundamente a notação Big O e documentar a complexidade no cabeçalho de cada script.
* **Medição:** Ter 100% dos exercícios entregues até a Prova 1 com a análise de complexidade correta.
* **Prazo:** Data da Prova 1 (P1).

### Meta 2: Aplicação Prática de Programação Dinâmica
* **Contexto:** Reduzir o custo computacional em modelos de análise financeira repetitivos.
* **Descrição:** Resolver problemas de otimização utilizando técnicas de memoização e tabelas.
* **Plano de ação:** Praticar 2 exercícios semanais de DP em plataformas como LeetCode ou Beecrowd focando nos temas da P1.
* **Medição:** Conseguir implementar uma solução de DP do zero durante a Prova 1 sem auxílio externo.
* **Prazo:** Data da Prova 1 (P1).

### Meta 3: Domínio em Algoritmos de Grafos
* **Contexto:** Mapear fluxos de trabalho e dependências de processos de clientes.
* **Descrição:** Implementar um algoritmo de caminho mínimo (Dijkstra) funcional para resolver problemas de conectividade.
* **Plano de ação:** Estudo dirigido de grafos e implementação de casos de uso reais aplicados a processos.
* **Medição:** Finalizar o semestre com um protótipo funcional capaz de processar um grafo de pelo menos 10 nós.
* **Prazo:** Final do semestre (Junho/2026).

Pretendo usar a IA como um **tutor pedagógico**.
* **Como:** Para explicar conceitos teóricos complexos e sugerir casos de teste (inputs/outputs) para validar meus códigos.
* **Limites:** Não utilizarei a IA para gerar soluções completas (copy-paste). Caso use a IA para entender um erro de lógica, deverei ser capaz de explicar a correção linha por linha, garantindo que o aprendizado foi efetivo para a avaliação presencial.
